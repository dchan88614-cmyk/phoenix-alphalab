## Part 1: Add Trade Simulator

Create:

`src/trading/trade_simulator.py`

For each historical BUY decision, simulate a trade using next-day or EOD-compatible assumptions:

- Signal date: date generated by EOD system.
- Entry date: next trading day after signal date.
- Entry price: next trading day open if available; otherwise next trading day close.
- Stop loss: entry_price - 1.5 * ATR from signal date, if ATR available.
- Fallback stop loss: entry_price * 0.92.
- Target 1: entry_price + 2 * (entry_price - stop_loss).
- Target 2: entry_price + 4 * (entry_price - stop_loss).
- Max holding period: 20 trading days.

Path logic:

- Use daily high/low after entry to determine stop/target hits.
- If stop and target are both hit on the same day, assume worst-case: stop hit first.
- Exit rules v0:
  - If stop hit: exit at stop_loss.
  - Else if target_2 hit: exit at target_2.
  - Else if target_1 hit: exit at target_1.
  - Else exit at close on max holding day.

Record per trade:

- signal_date
- entry_date
- ticker
- entry_price
- stop_loss
- target_1
- target_2
- exit_date
- exit_price
- exit_reason: STOP / TARGET_1 / TARGET_2 / TIME_EXIT
- holding_days
- realized_return
- benchmark_return_same_period
- realized_excess_return

## Part 2: Enforce Real Selectivity

The previous `max_buy_rate` only failed candidates after the fact. It did not actually reduce trades.

For v0.2, candidate parameters must actively control BUY frequency.

Add candidate parameters:

- max_buy_rate: 1.0, 0.7, 0.5, 0.3, 0.15
- min_relative_volume_prev20: 1.0, 1.25, 1.5, 2.0
- min_smoke_score: 0.70, 0.75, 0.80, 0.85
- min_rank_gap: 0.00, 0.03, 0.05, 0.08
- require_return_5d_positive: true/false
- require_return_20d_positive: true/false
- distance_to_52w_high_prev_min: -0.35, -0.25, -0.15
- dollar_volume_min: 10,000,000 / 20,000,000 / 50,000,000

Rank gap:

- For each signal date, compute top candidate smoke_score minus second candidate smoke_score.
- BUY only if rank_gap >= min_rank_gap.

Max BUY rate implementation:

- This must be active, not only a reporting gate.
- For each candidate rule, first generate all eligible BUY dates.
- If eligible BUY rate exceeds max_buy_rate, keep only the top signal dates by decision strength until max_buy_rate is reached.
- Decision strength may be:
  - smoke_score
  - rank_gap
  - relative_volume_prev20
  - simple average of normalized values
- Do not use forward returns or realized returns to choose which dates to keep.

## Part 4: Research-Qualified Gate v0.2

A candidate is `RESEARCH_QUALIFIED_NOT_LIVE` only if all are true:

1. At least 8 windows have sufficient data.
2. Final BUY count >= 40 across all windows.
3. Final overall BUY rate <= 50%.
4. Overall average realized excess return > 0.
5. Overall realized win rate >= 52%.
6. At least 6 valid windows have positive realized excess return.
7. Removing the single best trade still leaves average realized excess return > 0.
8. Worst realized return must be greater than -25%.
9. Stop hit rate must be <= 65%.

If it fails any gate, label:

`RESEARCH_ONLY_NOT_TRADABLE`
